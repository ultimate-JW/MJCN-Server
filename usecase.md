## 1. 회원가입 / 인증

### UC-1.1 이메일 회원가입

| 항목 | 내용 |
|------|------|
| 액터 | 비인증 사용자 |
| 사전조건 | 없음 |
| 사후조건 | 사용자 계정 생성, 이메일 인증 대기 상태 |

**기본 흐름**
1. 사용자가 이메일을 입력한다.
2. 사용자가 비밀번호를 입력한다.
3. 사용자가 비밀번호 확인을 입력한다.
4. 시스템이 유효성을 검사한다.
5. 시스템이 입력한 이메일로 인증 코드를 발송한다 (유효기간 3분).
6. 사용자가 인증 코드를 입력한다.
7. 시스템이 인증 코드를 검증한다.
8. 이메일 인증이 완료된다.

**대안 흐름**
- 4a. 이메일 형식 불일치 → "이메일 형식을 확인해주세요." 표시
- 4b. 중복 이메일 → "이미 사용 중인 이메일입니다." 표시
- 4c. 비밀번호 조건 미충족 (8~20자, 영문+숫자+특수문자) → 에러 메시지 표시
- 4d. 비밀번호 확인 불일치 → "비밀번호가 일치하지 않습니다." 표시
- 7a. 인증 코드 불일치 → "인증 코드가 일치하지 않습니다." 표시
- 7b. 인증 코드 만료 → "인증 코드가 만료되었습니다. 다시 요청해주세요." 표시
- 7c. 사용자가 "인증 코드 다시 보내기"를 선택 → 새 인증 코드 발송

### UC-1.2 프로필 설정 (온보딩)

| 항목 | 내용 |
|------|------|
| 액터 | 인증 사용자 |
| 사전조건 | UC-1.1 이메일 인증 완료 |
| 사후조건 | 사용자 프로필 설정 완료, 메인화면 이동 |

**기본 흐름**
1. [Step 1] 사용자가 기본 정보를 입력한다 (이름, 학년, 학기, 졸업 희망 시기(선택)).
2. [Step 2] 사용자가 전공을 선택한다 (대학 → 학부/학과 → 전공, 3뎁스).
3. [Step 3] 사용자가 관심분야를 선택한다 (최소 1개, 최대 3개).
4. [Step 4] 사용자가 수강이력을 입력한다 (선택, 건너뛰기 가능).
5. [Step 5] 사용자가 현재 수강과목을 입력한다 (선택, 건너뛰기 가능).
6. 회원가입 완료 화면이 표시된다.
7. 사용자가 "시작하기"를 누르면 메인화면으로 이동한다.

**대안 흐름**
- 1a. 이름 조건 미충족 (한글/영어, 2~10자) → 에러 메시지 표시
- 2a. 단일학부/학과인 경우 → 전공 선택 스텝 스킵
- 4a. 사용자가 "건너뛰기" 선택 → Step 5로 이동
- 5a. 사용자가 "건너뛰기" 선택 → 완료 화면으로 이동

### UC-1.3 이메일 로그인

| 항목 | 내용 |
|------|------|
| 액터 | 비인증 사용자 |
| 사전조건 | 회원가입 및 이메일 인증 완료 |
| 사후조건 | JWT 토큰 발급, 메인화면 이동 |

**기본 흐름**
1. 사용자가 이메일과 비밀번호를 입력한다.
2. 시스템이 자격 증명을 검증한다.
3. 시스템이 JWT 토큰(access + refresh)을 발급한다.
4. 메인화면으로 이동한다.

**대안 흐름**
- 2a. 이메일 또는 비밀번호 불일치 → "이메일 또는 비밀번호가 올바르지 않습니다." 표시
- 2b. 이메일 미인증 상태 → "이메일 인증을 완료해주세요." 표시

### UC-1.4 카카오 로그인

| 항목 | 내용 |
|------|------|
| 액터 | 비인증 사용자 |
| 사전조건 | 카카오 계정 보유 |
| 사후조건 | JWT 토큰 발급 |

**기본 흐름**
1. 사용자가 카카오 로그인 버튼을 누른다.
2. 프론트에서 카카오 SDK로 인가 코드를 획득한다.
3. 백엔드에 인가 코드를 전달한다.
4. 시스템이 카카오 API로 사용자 정보를 조회한다.
5. 기존 회원인 경우 JWT 발급 후 메인화면 이동.

**대안 흐름**
- 5a. 최초 로그인 → 자동 회원가입 처리 후 온보딩(UC-1.2)으로 이동
- 5b. 카카오 이메일과 기존 이메일 계정 동일 → 계정 연동 처리

### UC-1.5 로그아웃

| 항목 | 내용 |
|------|------|
| 액터 | 인증 사용자 |
| 사전조건 | 로그인 상태 |
| 사후조건 | refresh 토큰 무효화, FCM 토큰 삭제 |

**기본 흐름**
1. 사용자가 로그아웃을 선택한다.
2. 시스템이 refresh 토큰을 블랙리스트에 등록한다.
3. 시스템이 FCM 디바이스 토큰을 삭제한다.
4. 로그인 화면으로 이동한다.

### UC-1.6 비밀번호 재설정

| 항목 | 내용 |
|------|------|
| 액터 | 비인증 사용자 |
| 사전조건 | 이메일로 가입한 계정 존재 |
| 사후조건 | 비밀번호 변경 완료 |

**기본 흐름**
1. 사용자가 "비밀번호를 잊으셨나요?"를 선택한다.
2. 사용자가 가입 이메일을 입력한다.
3. 시스템이 인증 코드를 발송한다 (유효기간 3분).
4. 사용자가 인증 코드를 입력한다.
5. 사용자가 새 비밀번호 + 비밀번호 확인을 입력한다.
6. 비밀번호 변경 완료 후 로그인 화면으로 이동한다.

**대안 흐름**
- 3a. 미가입 이메일 → 동일한 발송 완료 안내 표시 (계정 존재 여부 노출 방지)
- 3b. 카카오 전용 계정 → "카카오 로그인으로 가입된 계정입니다." 표시
- 4a. 인증 코드 불일치/만료 → 에러 메시지 표시
- 5a. 비밀번호 조건 미충족 → 에러 메시지 표시

## 2. AI 비서 - 띵똥이

### UC-2.1 AI 채팅 (텍스트)

| 항목 | 내용 |
|------|------|
| 액터 | 인증 사용자, AI |
| 사전조건 | 로그인 상태 |
| 사후조건 | 채팅방 생성/업데이트, 메시지 저장 |

**기본 흐름**
1. 사용자가 메시지를 입력하여 전송한다.
2. 시스템이 새 채팅방을 자동 생성한다 (첫 메시지인 경우).
3. AI가 첫 메시지 기반으로 채팅방 제목을 자동 생성한다.
4. AI가 사용자 프로필 + 대화 히스토리 + 학교 데이터를 컨텍스트로 응답을 생성한다.
5. 응답이 저장되고 사용자에게 반환된다.

**대안 흐름**
- 2a. 기존 채팅방에서 메시지 전송 → 새 채팅방 생성 없이 대화 이어서 진행

### UC-2.2 AI 채팅 (첨부파일)

| 항목 | 내용 |
|------|------|
| 액터 | 인증 사용자, AI |
| 사전조건 | 로그인 상태 |
| 사후조건 | 첨부파일 저장, AI 응답 생성 |

**기본 흐름**
1. 사용자가 파일을 첨부하여 전송한다 (이미지/동영상/파일).
2. 시스템이 파일 형식 및 크기를 검증한다 (jpg/png, mp4, pdf/docx, 10MB 이하).
3. 파일이 업로드되고 AI에 전달된다.
4. AI가 응답을 생성한다.

**대안 흐름**
- 2a. 지원하지 않는 형식 또는 10MB 초과 → 에러 메시지 표시

### UC-2.3 맞춤형 PUSH 알림 추천

| 항목 | 내용 |
|------|------|

| 사전조건 | 사용자 프로필 설정 완료, 콘텐츠(공지/공모전) 존재 |
| 사후조건 | 관련 사용자에게 PUSH 알림 전송 |

**기본 흐름**
1. 트리거 발생 (새 공지 등록 / 마감 3일 전 / 마감 전날).

3. 관련도 상위 3개 이상 콘텐츠를 선별한다.
4. 해당 사용자에게 PUSH 알림을 전송한다.

## 3. 수강/졸업 관리

### UC-3.1 다음학기 수강과목 추천

| 항목 | 내용 |
|------|------|
| 액터 | 인증 사용자 |
| 사전조건 | 프로필(전공, 수강이력) 설정 완료 |
| 사후조건 | 추천 과목 리스트 반환 |

**기본 흐름**
1. 사용자가 다음학기 수강과목 추천을 요청한다.
2. 시스템이 수강이력 + 전공 + 학년/학기를 기반으로 분석한다.
3. 졸업요건 충족, 선후수 과목, 이수구분 균형을 고려하여 추천한다.
4. 전공(필수/선택) / 교양(필수/선택)으로 분리하여 결과를 반환한다.

---

### UC-3.2 전체 커리큘럼 추천

| 항목 | 내용 |
|------|------|
| 액터 | 인증 사용자 |
| 사전조건 | 프로필(전공, 수강이력, 졸업 희망 연도) 설정 완료 |
| 사후조건 | 졸업까지 학기별 추천 과목 리스트 반환 (2~5안) |

**기본 흐름**
1. 사용자가 전체 커리큘럼 추천을 요청한다.
2. 시스템이 수강이력 + 전공 + 졸업 희망 연도를 기반으로 분석한다.
3. 최소 2안, 최대 5안의 커리큘럼을 제시한다.
4. 각 안을 학기별, 이수구분별로 분리하여 반환한다.

## 4. 통합 정보 제공

### UC-4.1 공지사항 조회

|------|------|

## 5. 채팅방 보관함

### UC-5.1 채팅방 관리

| 항목 | 내용 |
|------|------|
| 액터 | 인증 사용자 |
| 사전조건 | 로그인 상태, 채팅 기록 존재 |
| 사후조건 | 채팅방 조회/삭제 |

**기본 흐름**
1. 사용자가 채팅방 보관함에 진입한다.
2. 전체 채팅방 목록이 최신순으로 표시된다 (제목, 마지막 메시지 미리보기, 날짜).
3. 사용자가 채팅방을 선택하면 대화를 이어서 진행한다.

**대안 흐름**
- 2a. 사용자가 카테고리 필터 선택 → 폴더별 조회 (공지/공모전/취업·진로/취미/기타)
- 3a. 사용자가 삭제를 선택 → 채팅방 삭제
